# 수익성·강건성 백테스트 — 사전등록 설계 (2026-06-23)

## 0. 목적 (이전 백테스트와 구분)

이전 두 백테스트는 **"시스템에 결함이 없다"**(소표본 존재증명)를 검증했고, 4개 이슈 전부
"단일층 결함 아님"으로 결론났다. 이 백테스트의 목적은 다르다 — **수익성·강건성 측정**.
핵심 질문은 "상승기에 얼마 버나"가 아니라 **"하락·횡보기에 손실을 얼마나 억제하나"**
(§3.5 시장 게이트의 실효성).

명시적 구분:
- **이전** = 결함 부재 검증, 큐레이션된 8종목(유형 대표) → 선택 편향 내재.
- **이번** = 수익성·강건성 측정 → **대표성 표본·편향 차단**이 최우선.

### 진입일-기준 국면 라벨의 한계 (명시)
국면 라벨은 **진입 시점(의사결정 시점)**의 시장 상태로 붙인다 — 게이트는 진입 시점에
작동하므로 인과적으로 옳은 선택이다. 그러나 **보유기간 중 국면이 바뀔 수 있다**(예:
하락기 진입 후 회복기에 청산). 따라서 "하락기 트레이드"는 *하락기에 진입한* 트레이드이지
*보유 내내 하락기였던* 트레이드가 아니다. 국면 경계 근처 트레이드의 초과수익 해석 시
이 점을 감안한다(보유기간 가중 국면 분해는 범위 밖).

## 4. 트리거·청산 (기존 결정론 엔진 재사용)

- `kr_pipeline/backtest/`의 `trigger_gate.evaluate`(진입) + `trigger_sim.simulate`(루프)
  그대로. **드라이버만** (100종목 × 2021–2024 × 국면 조인)으로 확장.
- 진입 = 트리거일 종가, 청산 = invalidation(price-only: `stop_loss`(base_low) 또는
  `sma_50` 이탈, 거래량 미고려 = 보수적), 재진입 = 새 pivot 갱신 후만(같은 pivot 재진입
  금지), look-ahead 차단(fresh_cross `prev_close`, 날짜 정렬·≤index). adj_volume 사용
  (gate `avg_volume_50d`와 단위 일치).
- **결정론 근사 = 하한 추정**: LLM 트리거 확인·포지션 사이징 제외(전수 비용상).

### 결정론 트리거의 시장-비게이팅 (중요 — 보수적 방향)
`trigger_gate.evaluate`는 `classification`/`watch_reason`은 받지만 **market_status는 받지
않는다**. `breakout_from_watch`(watch pivot 돌파)는 시장 국면을 재검사하지 않고 발화한다.
즉 하락기에도 watch pivot이 돌파되면 결정론 시뮬은 진입한다. 실제 시스템은 LLM 트리거
확인 단계에서 §3.5를 재적용하지만 이 백테스트는 그것을 제외한다 →
**결정론 근사는 하락기 진입을 과대평가**(게이트 방어를 과소평가, 손실을 과대평가)하는
**보수적** 방향이다. 따라서 측정된 방어력은 실제 시스템의 하한이다.

게이트 방어의 *순수* 측정은 분류층에서 직접 한다(§6, entry-classification rate).

## 5. 멱등 resume (기존 backfill 패턴 확장)

- 분류 적재 = `classification_backfill` 테이블(PK `(symbol, analyzed_for_date)`,

- 시작 전 (샘플 100종목의 실제 qualifying 주간점 합)으로 총 호출 예상치를 산출·기록.

## 6. 채점 (국면별 분리가 핵심 산출)

- **1차 지표 = 시장대비 초과수익**(`trigger_sim.market_relative`, 보유기간 지수수익
  차감). 절대수익은 국면에 오염돼 국면 비교 불가 → 시장대비가 1차.

- **보조(국면별)**: 트리거 발화 수, 승률, 평균 손익비, 최대 낙폭, 트레이드 수.

## 7. 사전등록 성공/실패 기준 (결과 보고 전 박음)

### 7.1 게이트 방어 — 분류층 직접 측정 (1차, 트리거 근사와 무관하게 깨끗)
게이트의 본질은 "하락·조정기에 `entry` 분류를 억제"하는 것(§3.5 hard rule:
downtrend/correction/unconfirmed_rally에서 `entry` 금지). 결정론 트리거의
`breakout_from_watch` 누수(§4)와 무관하게 분류층에서 직접 측정:

- **entry-classification rate(phase)** = (해당 국면 분류점 중 `classification=='entry'` 수)
  / (해당 국면 전체 분류점 수). 국면 = 분류 기준일(금요일)의 시장 상태.
  - **분모 = 이번 100종목의 분류점**(~1,838개)을 국면별로 나눈 것. 이 지표는 **분류점
    기준**이라 §7.5 트레이드-수 가드(트레이드 기준)와 **단위가 별개**다 — 분류점은
    많아도 트레이드는 적을 수 있다(역도 성립).
- **사전등록 성공 기준**: `R_down = entry-rate(downtrend ∪ correction)`,
  `R_up = entry-rate(confirmed_uptrend)` 에 대해 **`R_down / R_up ≤ 0.5`**
  → 게이트가 하락·조정기 진입 분류를 유의하게(상승기의 절반 이하로) 억제 = **방어 입증**.
  hard rule이 작동하면 `R_down`은 0에 가까울 것으로 기대(0 초과분은 경계일·라벨 불일치).
- **실패/재검토**: `R_down / R_up > 0.5` → 게이트가 진입 분류를 충분히 억제하지 못함,
  §3.5 재검토.

### 7.2 손실 억제 — 시장대비 초과수익 (보조)
- **사전등록 기준**: `(downtrend ∪ correction)` 국면 트레이드의 **평균 시장대비
  초과수익 ≥ 0** 이면 방어 보강 근거.
- **위상**: §7.1이 **1차(깨끗)** — 분류층 직접 측정이라 트리거 근사에 오염 안 됨.
  §7.2는 **보조** — 이 트레이드 표본은 §4 결정론 트리거 누수(`breakout_from_watch`
  시장 비게이팅)의 영향을 받아 **하락기 억지 진입을 포함**한다.
- **오판 방지**: **§7.2 음수가 곧 게이트 실패는 아니다** — 시장 비게이팅 트리거가
  강제한 진입이 손실을 끌어내린 것일 수 있으므로, 게이트 재검토는 §7.1(분류층)로
  판정한다. §7.2 음수는 "결정론 근사의 하한 손실"로 해석한다.

### 7.3 상승 기준선 (2차, 정보용)
- confirmed_uptrend 평균 시장대비 초과수익 — 2024 결과와 비교(정보용, 합격선 없음).

### 7.4 횡보 whipsaw (2차, 정보용)
- rally_attempt 국면 승률·평균손실 — 가짜 돌파로 잦은 손절을 까먹는지(정보용).

### 7.5 검정력 가드 (사전등록)
- 국면별 트레이드 < **10건**이면 그 국면 결론은 **underpowered로 표기**, 과해석 금지.
  특히 confirmed_uptrend는 표본일이 적어(KOSDAQ 93일) 트레이드가 적을 수 있음 — 그래도
  사후에 기준을 바꾸지 않는다.

## 8. 한계 (예상)

1. **결정론 트리거·청산 = 하한 추정**: LLM 트리거 확인·사이징 제외. 특히
   `breakout_from_watch`가 시장 비게이팅이라 하락기 진입을 과대평가(보수적, §4).
2. **입력 분류 = 1회 LLM 백필**: "그 입력 고정 하의" 결과. 비결정성 때문에 재실행 비교
   금지.
3. **생존 편향 잔존**: 상폐 종목 부재(DB `delisted_at` 전무) → 결과가 부풀려지는 방향의
   편향(§2).
4. **진입일-기준 국면 라벨**: 보유기간 중 국면 변동 미반영(§3).
5. **KR 단일 시장·특정 연도**: 그 시장·기간 특성에 의존(타 시장/기간 일반화엔 추가 표본).
