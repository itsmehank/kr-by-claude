## 1. 대상 가설 (감사 보고서 §H1/H2 그대로)

- **H1**: 5b 합격선(거래량 1.4× + 상단 1/3 마감 + 종가>pivot 동시 요구)이 과엄격.
- **H2**: 결정론 게이트(ALLOWED_WATCH_REASONS 제한 + fresh_cross + 1.0× 거래량)가
  재돌파 기회를 구조적으로 차단.

## 2. 1부 — H1 이득 측정 (LLM 필요, 승인 후 분할 실행)

- **셀 동결**: `data/backtest/recall_funnel_20260705.csv`(리포 커밋본 자체가 동결
  파일) 의 `gate_anyday_fired` 비어있지 않은 **38건** — as_of = 게이트 발화일
  (측정 왜곡 없는 조건 — 원 감사의 장중 첫 터치일 교락 제거).
- 각 셀 1 LLM 호출(5b 재생 `recall_phase2.py` 재사용, **실행 전 확장 필요**:
  요청 payload 의 `computed_gates` 를 호출별로 영속 — 현행은 decision 4필드만
  저장, 검증 리뷰 실측). 완화선 재호출 금지(LLM 비결정성 교락 방지).
- **완화선 재판정 규칙(사전 고정 — 결정론 추론)**: 완화선(거래량 1.4→**1.2**,
  BREAKOUT_VOL_WAIT_FLOOR 준용) go_now =
  현행 go_now ∪ (LLM decision==wait ∧ payload 기록 `volume_band=='wait_band'`
  ∧ 나머지 §3.1 go_now 게이트 전부 pass).
  정직 명시: 이 완화선 결과는 payload 기록 기반 **결정론 추론**이지 LLM 의 실제
  판정이 아니다 — 1.2~1.4× 구간에서 LLM 재량이 개입했을 여지는 재호출 없이는
  알 수 없고, 재호출 금지가 우선이므로 이 한계를 안고 측정한다(보수 해석:
  완화 이득의 상한).
- 지표: 합격선별 go_now 수(38 중). 이득 = 완화선 go_now − 현행 go_now.

## 3. 2부 — 비용 측정: 패자 대조군 (LLM 필요, 승인 후 분할 실행)

- **대조군 구성 규칙(결정론, 실행 시 동결 파일 생성)**: **모집단 = 표본 B
  (`backtest_classification`, 독립 100종목 — 1층 분류가 이미 존재해 대조군 구성이
  결정론 쿼리로 완결, 사용자 결정 2026-07-22)** 의 watch/entry 셀 중 ①pivot 有
  ②이후 피벗 상회 돌파일 존재 ③돌파 후 12주 수익률 < 0 (패자) — 표본 상한
  40건(1부와 규모 균형). 표본 B 에서 미달 시에만 recall 감사 창의 타 종목으로
  보충. 세부 쿼리는 실행 시 본 문서 부록 B 로 동결 후 착수.
- 각 셀 1 LLM 호출, 1부와 동일한 이중 합격선 판정.
- 지표: 합격선별 대조군 go_now 수 = 나쁜 신호 유입 비용.

## 4. H2 — 결정론 게이트 완화 시뮬 (LLM 0회, 본 문서와 함께 즉시 실행)

- **모집단**: `recall_audit_classification` 의 watch/entry & pivot 有 = **217 셀**
  (전원 승자 에피소드의 주간 앵커 — 비용 측면은 측정 불가, **이득 상한**만 측정.
  비용은 2부 대조군에 완화 게이트를 동일 적용해 측정 — 승인 후).
- **재생 규약**: 각 셀의 analyzed_for_date 이후 5거래일(다음 주말 재분류 전),
  `trigger_gate.evaluate` 현행 코드로 일별 판정. 입력 = daily_prices(close,
  volume, prev_close) + daily_indicators(avg_volume_50d, sma_50), stop_loss =
  base_low(현행 load 규약 그대로).
- **변형(사전 고정)**:
  - V0 현행(기준선)
  - V1 watch_reason 제한 해제(ALLOWED_WATCH_REASONS → 전체 watch 사유 허용)
  - V2 fresh_cross 요구 해제(watch 도 entry 처럼 close>pivot 이면 발화 가능)
  - V3 = V1+V2
- **#45 정합**: 발화 중 `close > pivot × 1.05` 는 현행 파이프라인이 결정론 wait
  로 차단(LLM 0회) — 변형별 발화 수에서 이 차단분을 제외한 값을 **실 5b 호출
  증가분**으로 계상.
- 지표(변형별): 상향 발화 셀 수 / 그중 extended 차단 제외 실호출 수 / 참고 —
  발화일 +20거래일 수익률 중앙값(승자 표본 편향 명시 — 판정 근거 아님, 관찰만).

## 5. 판정 기준 (사전 고정)

- **완화 채택 조건**(합격선·게이트 각각 독립 판정):
  이득(1부: 완화로 늘어난 승자 go_now) > 비용(2부: 완화로 늘어난 대조군 go_now).
  동수이면 현행 유지(보수 기본값).
- H2 게이트 완화는 위 부등식에 **추가 5b 호출 비용**(H2 시뮬 실호출 증가분 ×
  대략 호출당 비용)을 비용 측에 합산 — 신호 품질과 무관한 순수 운영 비용.
  단 H2 시뮬의 실호출 단위는 **셀**(첫 발화 기준) — V2/V3 는 fresh_cross 해제로
  같은 셀이 여러 날 재발화하므로 셀 수는 실제 호출 수의 **하한**(검증 리뷰 지적,
  보수 방향 아님을 명시 — 운영 비용은 이보다 크다).
- 어느 변형도 조건 미충족 시 현행 유지로 #2 종결.
