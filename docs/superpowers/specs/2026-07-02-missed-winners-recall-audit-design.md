# 미포착 승자 recall 감사 (일회성) — 설계 (2026-07-02)

## 0. 성격 · 목적

- **recall 감사**: 수익성 백테스트(precision — *잡은 것*이 좋았나)의 상보. 이 감사는
  *좋은 것(대상승 종목)을 잡았나* 를 측정한다.
- **탐색형(가설 생성)**: 확증형 백테스트와 달리 PASS/FAIL 판정이 목적이 아니다.
  **모든 발견은 가설이며, prompt/룰 변경은 별도 패널 대조 검증(완화 시 유입되는
  false-positive 비용 측정 — SK 검증과 동일 절차)을 통과한 후에만 적용한다.**
  감사 → 가설 → 검증 → 변경 4단계를 건너뛰지 않는다.
- 목적 2건 (사용자 원안):
  1. LLM 분류가 놓친 종목 중 "놓치면 안 됐던" 것의 존재·원인 규명 → 시스템 개선 입력.
  2. "entry/actionable 기준이 과하게 높은 것 아닌가" 우려의 실증 판별. 단 측정 지표는
     주말 entry 빈도가 아니라 **actionable coverage** (대상승 전 watch+유효 피벗 보유
     + 트리거 go_now 여부) — 실매수 경로가 watch→평일 트리거이기 때문.
- **감사 대상 = 현재 시스템**: 백필은 현행 prompt(`analyze_chart_v3.md`, verdict 규율
  A 반영본)로 실행한다. 질문은 "지금 시스템을 그 시점에 두면 놓치는가"이다.

## 3. Phase 0 — 승자 선정 + 결정론 귀속 (LLM 0회)

### 3.1 승자 후보 (주 단위 스캔)

- anchor: 감사 창의 매주 토요일. 데이터는 그 주 **금요일(주 마지막 거래일)** 종가까지.
- 수익률: **adj_close 기준** 후행 13주·26주 수익률. 시장 초과분 = 종목 수익률 −
  소속 시장 지수(KOSPI/KOSDAQ composite) 동일 구간 수익률.
- 선정: 13주 초과분 ≥ **T13** 또는 26주 초과분 ≥ **T26** — *T13·T26 은 §7 의 2차
  잠금 대상* (잠정 T13=+30%p, T26=+50%p). 단 **해당 창의 절대 수익률 > 0 필수**
  (하락장에서 "덜 빠진" 상대 승자 배제 — 추세추종 감사 대상이 아님).
- 유동성 하한: **첫 TT 통과일(§3.3) 전후 20거래일 평균 거래대금(`daily_prices.value`)
  ≥ L** — *L 은 2차 잠금 대상* (잠정 10억원). 급등 중 거래대금 폭증 때문에 측정
  시점을 "시스템이 처음 봤을 시점"으로 고정한다. TT 통과가 없는 에피소드(=필터
  배제 버킷)는 에피소드 시작주 기준으로 측정. **적용 순서: §3.3 의 TT 판정 후**
  (측정 기준일이 §3.3 산출물에 의존).
- 총 상한: 최종 에피소드 수 > **N_cap** 이면 **초과수익 터사일(3분위) 층화 균등
  표집**(각 분위 ⌈N_cap/3⌉ 무작위, seed=20260702) — *N_cap 은 2차 잠금*.
  상위 절단 금지: 상위-N 절단은 표본을 극단 급등주(climax/extended 성격)로
  편향시켜 coverage 헤드라인이 실제보다 나쁘게 나온다.

### 3.2 에피소드 병합 (1차 잠금 확정)

- 같은 종목의 승자 당첨 주가 **간격 ≤ 4주**로 이어지면 동일 에피소드.
- 에피소드 시작 = 첫 당첨 주의 13주(해당 창) 시작점, 고점 = 에피소드 내 adj 최고 종가.

### 3.3 결정론 귀속 1층 (1차 잠금 확정)

- **TT 판정은 금요일 기준**: 그 주 마지막 거래일의 `daily_indicators.minervini_pass`.
  주말 러너의 실제 게이트(`llm_runner/load.py` — `minervini_pass=TRUE` 단일 조건)와
  동일 — 주중 일시 통과는 시스템이 본 적 없으므로 집계하지 않는다.
- 에피소드 시작 −13주 ~ 고점 주 사이 금요일 TT 통과 주 **0** → 버킷 `filter_excluded`.
  LLM 없이 종결(개수 집계 + 상위 5건 육안 표본 확인).
- **잔여 상승분** = 첫 TT 통과 주 금요일 adj 종가 → 에피소드 고점 수익률.
  **< 20%** (1차 잠금 고정) → 버킷 `structurally_uncatchable` — 완벽 분류·트리거로도
  먹을 게 없던 케이스. LLM 없이 종결.
- 잔여 에피소드 → Phase 1 대상. **백필 대상 주 = 첫 TT 통과 주부터 고점 주까지 중
  금요일 TT 통과 주 전부, 에피소드당 상한 12주(초과 시 앞에서부터 12주 — 진입 기회는
  초기에 있다)** (1차 잠금 고정). 상한 절단 발생 시 **truncated=TRUE 플래그** —
  truncated 에피소드의 `classification_miss` 는 후기 베이스(2차 이후)가 12주 이후
  형성됐을 가능성(미관측) 때문에 §9 보고서에서 **"신뢰도 낮음" 별도 표기**.

### 3.4 데이터 품질 가드

- `corporate_actions` 매핑이 있는 종목은 에피소드에 플래그 → 상위 수익률 10건 육안
  검증(역분할 유령 승자 차단). adj 계열 이상(0/NULL) 검출 시 해당 에피소드 격리 표시.
- 실행 전 ohlcv/indicators 증분을 2026-06-30까지 최신화.

## 5. Phase 2 — 트리거층 귀속 (거의 결정론)

- Phase 1 결과 watch 행의 `pivot_price`: 에피소드 내 이후 거래일에서 adj 고가 ≥ pivot
  돌파 여부 결정론 확인 + production `trigger_gate` 결정론부 재적용.
- 돌파 건: 트리거 확인 LLM 감사 — 7/2 사전등록 §1 과 동일 방식(`build_for_5b`
  prior_row 주입, `evaluate_pivot_trigger_v1.md`, as_of=돌파일). 저장은
  `data/backtest/recall_trigger_audit_20260702.json` 파일(기존 관례).
- 에피소드 최종 버킷 (우선순위 판정, 1차 잠금 고정):
  1. `caught` — Phase 1 에 entry 존재, **또는** watch+실돌파+go_now.
  2. `trigger_miss` — watch+실돌파+wait/abort.
  3. `pivot_miss` — watch 존재하나 pivot 미돌파(피벗 산정 문제 후보).
  4. `classification_miss` — 백필 주 전부 ignore.
  (+ Phase 0 의 `filter_excluded` / `structurally_uncatchable` / `censored`)

## 6. 딱지 정확도 채점 (개선-가설 승격 게이트)

`classification_miss` 의 각 ignore 주에 대해 **결정론 채점 컬럼**을 붙인다
(SK expert-inquiry `A_weekly_classification_vs_indicators.csv` 방법론 재사용):

- `climax_run` 플래그 → prompt 정량 정의 충족 여부(max(1,2,3주 수익률) ≥ 25% +
  spread/volume 요소).
- `extended_from_ma` → SMA50 이격 > 15% 실측.
- 기타 정량화 가능한 flag 동일 방식.
- **RS 상태 컬럼**: 각 ignore 주의 시스템 RS rating(대 KOSPI 단일 벤치마크) 기록.
  승자 선정(§3.1)은 *소속 시장 지수* 대비 초과수익인데 시스템 RS 는 KOSPI 단일
  기준 — KOSDAQ 승자의 ignore 가 프롬프트 결함인지 벤치마크 설계 선택의 결과인지
  분리하기 위함.

**정의-위반 ignore 만 결함 가설로 승격한다.** 정의-충족 ignore 는 "정당한 보수성"으로
분류하고 가설로 올리지 않는다. 이 게이트가 누출(§8)·비결정성과 무관한 유일한
판정축이다.

## 7. 2단계 잠금 절차 (선정 파라미터)

1. 본 문서 커밋 = **1차 잠금**: 규칙 구조 전부 + §3.2/3.3/5 의 고정값(병합 4주,
   잔여 20%, 백필 상한 12주, 버킷 우선순위).
2. Phase 0 실행(무료·결정론) → 임계별 에피소드 수 분포 보고.
3. **T13 / T26 / L / N_cap 확정 → 본 문서에 추기 커밋 = 2차 잠금.** 조정은 총량
   파라미터만 — **종목 단위 개입 금지**(마릿수 정보만으로 결론을 조작할 유일한 통로).
4. 확정 호출량(Phase 1 = 백필 셀 수 정확값, Phase 2 = 상한) + 예상 소요(실측
   144초/호출 × 병렬) 보고 → **사용자 승인 후 LLM 첫 호출**.
5. LLM 첫 호출 이후 모든 기준 불변. 위반 = 감사 무효.

정당성: Phase 0 산출물(마릿수·수익률 분포)에는 결론(포착/미포착) 정보가 없다 —
분류가 아직 존재하지 않으므로. 결론 생성 시작점(LLM 첫 호출)이 잠금선이다.

## 8. 해석 규율 — LLM 사후지식 누출

- 모델 지식 컷오프(2026-01) 상 감사 창의 결과를 "알" 수 있다. **누출 방향은
  양방향 모두 가능**: 대상승을 알면 관대(포착 과대) / 이후 폭락까지 알면 보수(미포착
  과대). 따라서 "미포착 발견은 무조건 진짜" 식 비대칭 주장은 하지 않는다.
- 방어선: (a) 개별 판정 단위 해석 금지 — 패턴 수준만, (b) 결함 가설 승격은 §6 의
  **정의-위반 채점(결정론)** 통과 건만. 누출이 verdict 를 어느 쪽으로 밀었든
  정의-위반 여부는 오염되지 않는다.
