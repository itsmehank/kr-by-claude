## 1. 배경 및 목적

수집된 모든 데이터 (일/주봉 OHLCV, 지수, 지표, 거래량, 시장 컨텍스트, 기업행위) 를 사람이 탐색하고, LLM 에게 분석 요청을 보낼 수 있는 형태로 패키징하는 인터페이스 레이어 구축.

### 두 가지 목표

1. **사용자 인터페이스 (탐색)** — 5 페이지 React SPA. 히트맵으로 시장 전체 조망, 차트 페이지로 개별 종목 검증, 미너비니 통과 종목 페이지로 후보 발굴
2. **LLM 분석 입력 빌더 (ZIP)** — 종목별 통합 payload.json + 차트 이미지 + 프롬프트 파일 묶음. 사용자가 직접 ChatGPT/Claude.ai 에 붙여넣거나 #4 자동 분석이 직접 호출

### 핵심 원칙

- **`api/` 와 `web/` 독립** — `api/` 는 `kr_pipeline.db` 를 import 해서 DB 접근. `web/` 는 HTTP 만 알면 됨.
- **`api/services/` 가 비즈니스 로직 집중** — 라우터는 얇게. 각 builder 가 단일 책임.
- **`web/components/charts/` 는 React 래퍼** — Lightweight Charts 의 imperative API 를 React 컴포넌트로 감쌈.
- **타입 동기화 수동** — `api/schemas` Pydantic ↔ `web/src/lib/types.ts` TypeScript 양쪽 정의. openapi-codegen 자동화는 V2.

## 4. REST API 설계

### `/api/stocks/*`

```
GET  /api/stocks
     Query: ?market=KOSPI&sector=...&limit=100&q=삼성
     Response: [{ticker, name, market, sector, delisted_at}]

GET  /api/stocks/{ticker}
     Response: {ticker, name, market, sector, listed_at, delisted_at, is_common}
```

### `/api/indicators/*`

```
GET  /api/indicators/daily/{ticker}
     Query: ?start=&end=  (default: 최근 1년)
     Response: daily_indicators 행 배열 (V2 거래량 컬럼 포함)

GET  /api/indicators/weekly/{ticker}
     Query: ?start=&end=
     Response: weekly_indicators 행 배열

GET  /api/indicators/index/{index_code}
     Query: ?timeframe=daily|weekly&start=&end=
     Response: index_daily 또는 index_weekly 의 가격 시계열

GET  /api/indicators/minervini-passed
     Query: ?date=YYYY-MM-DD&min_rs=80
     Response: [{ticker, name, sector, rs_rating, adj_close, volume_ratio_50d, pocket_pivot_flag}]
```

### `/api/heatmap/*`

```
GET  /api/heatmap/sectors
     Query: ?date=YYYY-MM-DD
     Response: [{sector, stock_count, avg_rs_rating, minervini_pass_count, minervini_pass_rate}]

GET  /api/heatmap/sectors/{sector}/stocks
     Query: ?date=&min_rs=&max_rs=
     Response: [{ticker, name, rs_rating, adj_close, minervini_pass, volume_ratio_50d}]
```

### `/api/render/*` — 차트 PNG

```
GET  /api/render/{ticker}/daily.png
     Query: ?range_days=365&width=1400&height=900
     Response: image/png bytes

GET  /api/render/{ticker}/weekly.png
     Query: ?range_weeks=104&...
```

matplotlib 서버 사이드 렌더링. Lightweight Charts (브라우저) 와 별개.

### `/api/prompts/*` — LLM ZIP

```
GET  /api/prompts/{ticker}.zip
     Response: application/zip (~500 KB)
     ZIP 내용: §5 참조
```

### `/api/runs/*` — 운영 모니터링

```
GET  /api/runs
     Query: ?pipeline=&limit=20
     Response: pipeline_runs 최근 N개
```

### 공통 원칙

- REST 만 (GraphQL 안 함)
- 인증 없음 (로컬 단일)
- WebSocket / SSE 안 함 (실시간 갱신 불필요)
- 에러 형식: `{"detail": "..."}` (FastAPI 표준)
- 자동 OpenAPI 문서: `/docs`
- 페이지네이션 없음 (응답 작음 — 일봉 1년 ~200KB)

### `margin_pct` 계산 — 미너비니 조건별

LLM 의 "marginal pass" (< 3%) 판정 입력. 각 조건별로 의미가 다름:

```python
def margin_pct_c1(values):
    """close > sma_150 > sma_200 — 최소 chain margin."""
    return min(
        (values['close'] - values['sma_150']) / values['sma_150'] * 100,
        (values['sma_150'] - values['sma_200']) / values['sma_200'] * 100,
    )

def margin_pct_c2(values):
    """sma_150 > sma_200."""
    return (values['sma_150'] - values['sma_200']) / values['sma_200'] * 100

def margin_pct_c3(values):
    """sma_200(today) > sma_200(today-22)."""
    return (values['sma_200_today'] - values['sma_200_22d_ago']) / values['sma_200_22d_ago'] * 100

def margin_pct_c4(values):
    """sma_50 > sma_150 > sma_200 — 최소 chain."""
    return min(
        (values['sma_50'] - values['sma_150']) / values['sma_150'] * 100,
        (values['sma_150'] - values['sma_200']) / values['sma_200'] * 100,
    )

def margin_pct_c5(values):
    """close > sma_50."""
    return (values['close'] - values['sma_50']) / values['sma_50'] * 100

def margin_pct_c6(values):
    """close >= w52_low * 1.25."""
    threshold = values['w52_low'] * 1.25
    return (values['close'] - threshold) / threshold * 100

def margin_pct_c7(values):
    """close >= w52_high * 0.75."""
    threshold = values['w52_high'] * 0.75
    return (values['close'] - threshold) / threshold * 100

def margin_pct_c8(values):
    """rs_rating >= 70 — 절대 차이 (백분위 자체)."""
    return float(values['rs_rating']) - 70.0
```

음수 margin = 통과 안 함. 양수 작으면 marginal.

## 7. 에러 처리 / 사용자 경험

### 백엔드

- Pydantic 자동 검증 (422 응답)
- 존재 안 하는 ticker → 404 (`{"detail": "Stock not found"}`)
- DB 에러 → 500 (FastAPI 기본)
- 차트 렌더링 실패 (데이터 부족) → 200 with smaller chart 또는 placeholder PNG

### 프론트엔드

- TanStack Query `error` 상태 — 페이지에 "Failed to load" 메시지
- 로딩 → 스켈레톤 (shadcn 의 `Skeleton`)
- 404 → "종목 없음" 안내 + 검색 제안
- 네트워크 오류 → 재시도 버튼

### 의도적으로 안 하는 것

- 인증/인가 (로컬 단일)
- 권한 분기
- i18n (한국어 only)
- 다크모드 (구현 단계 frontend-design 결정)
- 모바일 반응형 (로컬 PC 가정)
- 실시간 알림

### 수동 스모크 (자율 실행 마지막)

자율 실행자가 사용자에게 안내:
1. 백엔드 + 프론트 dev 서버 띄움 (터미널 2개)
2. http://localhost:5173 접속
3. 5 페이지 클릭 — 데이터 표시 확인
4. 차트 페이지 종목 선택, 지표 토글
5. LLM 프롬프트 페이지 → ZIP 다운로드
6. ZIP 압축 풀어서 13 파일 확인
7. `payload.json` / `daily_chart.png` 열어보기

사용자 OK 신호 받으면 완료.

### TDD 적용 범위

- 백엔드 builder 함수들 → TDD 강력 권장 (순수 함수)
- API 라우터 → TestClient 로 통합 테스트
- React 페이지 → 구현 후 핵심 케이스만 (시각 검증 위주)
