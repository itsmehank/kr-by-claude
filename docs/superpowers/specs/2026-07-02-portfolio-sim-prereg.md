# 포트폴리오 단위 시뮬레이션 — 사전등록 (2026-07-02)

트레이드 단위 백테스트(`backtest-profitability-results.md`)의 후속: "1억을 넣고
4년 돌렸으면 계좌가 어떻게 됐나". **본 문서의 규칙은 결과 산출 전 고정.**
규칙 출처: Minervini/O'Neil 전문 AI 에이전트 답변 v1(7개 규칙) + v1.1(미결정 8건
해소, 결함수정 2건: 당일진입 교체순환·현금부족 트랜치) + 구현측 보완 3건(§4).
전부 결정론(LLM 0회), 일봉 종가 조건 → 당일 종가 체결.

## 1. 공통 설정

- 초기 자본 **1억 KRW**, 무레버리지 현금계좌, 수량은 통화 기준(소수 주 허용 — 한계 명시).
- 기간: 매매 2021-01-01 ~ 2025-06-30 (분류 신호는 ~2024-12, 이후는 잔여 active pivot).
- 신호: 기존 결정론 트리거(fresh cross + 거래량 ≥ 50일평균 + **5% 추격 상한**) —
  동결 100종목 backtest_classification watch 행 기반. 이미 보유 중 종목 신호 무시,
  같은 pivot 재진입 금지(기존과 동일).
- 비용: 매수/매도 수수료 각 0.015%, 매도 증권거래세 연도별(0.23/0.23/0.20/0.18/0.15%)
  — 거래 시점 차감(기존 사전등록 §2.2 승계).
- 벤치마크: **KOSPI(주)** + KOSDAQ(보조) 동일기간 buy&hold.

## 2. 규칙 (에이전트 v1.1 확정본)

- **사이징**: `stop_pct = (entry_close − max(base_low, sma50)) / entry_close`
  (base_low 부재 시 sma50 단독). `stop_pct > 10%` → 매수 스킵(건수·분포 보고,
  스킵률 >30% 면 상류 진단 신호). `position_pct = min(1.25%/stop_pct, 25%)`.
- **슬롯**: 최대 5 포지션.
- **교체**: 슬롯 만석 + 신규 신호 → 최약(평균매입가 대비 수익률 최저) 포지션이
  **≤ 0%** 일 때만 전량 매도 후 신규 진입. 교체 후보에서 제외: (i) 당일 진입
  (ii) 8주 면제 상태. 신호 우선순위 순 연쇄 허용(면제로 유한).
- **우선순위**: rs_rating DESC(결측=최하) → 돌파일 거래량배수 DESC.
- **본전 플로어(5A, 전 시나리오)**: 종가 ≥ 평균매입가×(1+3×stop_pct) 도달 시
  armed. armed 후 종가 < 평균매입가 → 전량 청산. 기존 청산(sma50/base_low)과 OR.
- **8주 면제**: 진입(+T1) 후 21 달력일 내 종가 ≥ 평균매입가×1.20 → 진입+56
  달력일까지 교체 면제.
- **피라미딩(S2·S3)**: 목표금액 = position_pct × T1일 계좌가치(§4-②, 원화 동결),
  트랜치 50/30/20. T2: 종가 ≥ T1가×1.02, T3: ≥ T1가×1.04 최초일. 소멸: (a) 트리거일
  종가 > pivot×1.05 (b) 포지션 청산 (c) 트리거일 현금 부족(부분체결 금지).
- **강세 분할매도(5B, S3만)**: 최초 종가 ≥ 평균매입가×1.20 일에 — 진입 후 >21일이면
  50% 매도, ≤21일이면 8주 면제 진입. 진입+56일 후 첫 거래일 종가 ≥ ×1.20 이면 50%
  매도, 미만이면 5B 만료(전량 보유, 5A만). 잔여분 = 5A + 추세 청산.
- **일중 순서**: ① 청산(스톱·플로어 — 상호순서 무관, 전량 동일) → ② 교체 매도 →
  ③ 매수. 매도 대금 당일 사용 가능.

## 3. 시나리오 매트릭스 (6곡선)

| | 하락기 진입 포함 | 하락기 진입 제외* |
|---|---|---|
| S1 기본(단일진입+5A) | S1-incl | S1-excl |
| S2 = S1+피라미딩 | S2-incl | S2-excl |
| S3 = S2+5B | S3-incl | S3-excl |

*제외 = 진입일 시장국면 ∈ {downtrend, correction} 인 **신규 진입(T1)** 신호 스킵 —
트리거 확인 감사(33/33 wait) 실측 반영. 피라미딩 추가분(T2/T3)은 포지션 추종
(감사 범위는 신규 진입이므로 — 한계 명시).

## 4. 구현측 보완 3건 (design-judgment, 에이전트 v1.1 이후 확정)

- **① 매수 단계 내 순서**: 피라미딩 트랜치 먼저, 신규 진입 나중 — "승자에 더한다"
  약속 이행 우선(O'Neil 피라미딩 취지).
- **② 목표금액 동결**: position_pct × **T1 진입일 종가 기준 계좌가치**를 원화로
  동결(이후 계좌 변동 무관) — 리스크 1.25% 공식의 소급 붕괴 방지.
- **③ 최약 판정 기준가**: 교체 규칙의 수익률 = 종가/가중평균매입가 − 1 (Q5 통일 연장).
- **④ T1 현금 부족**: 신규 진입(T1) 금액보다 자유 현금이 적으면 신호 전체 스킵
  (부분 체결 금지 — 트랜치 규칙 (c)와 동일 원칙, 건수 보고). 5슬롯×최대 25%는
  자본 초과 배분이 가능하므로 이 경우가 실제 발생한다.

## 5. 산출 지표 (사전 고정)

시나리오별: 최종 계좌배수, CAGR, **계좌 MDD**(일별 equity 피크 대비), 자본 가동률
(일평균 투자금/계좌가치), 진입 수·교체 수·슬롯부족 스킵 수·10%스톱 스킵 수(+분포)·
트랜치 소멸 수(사유별)·5B 매도 수. 벤치마크 대비 초과 CAGR.

**판정 성격**: 본 시뮬은 탐색적 측정 — 사전등록 대상은 규칙·지표이며 합격선은
두지 않는다(트레이드 층에서 이미 "시장초과 미입증"). 해석 시 주의 2건 고정:
(a) 표본 100종목은 실전 신호밀도의 ~1/16 (프레임 34,022 vs 표본 2,135 분류점) —
슬롯 경합·교체 규칙은 실전보다 훨씬 덜 발동하므로 **가동률·경합 수치는 하한**,
실전 일반화 불가. (b) 우측꼬리 소수 의존·생존편향 등 기존 §8 한계 전부 승계.
