# 포트폴리오 단위 시뮬레이션 — 사전등록 (2026-07-02)

트레이드 단위 백테스트(`backtest-profitability-results.md`)의 후속: "1억을 넣고
4년 돌렸으면 계좌가 어떻게 됐나". **본 문서의 규칙은 결과 산출 전 고정.**
규칙 출처: Minervini/O'Neil 전문 AI 에이전트 답변 v1(7개 규칙) + v1.1(미결정 8건
해소, 결함수정 2건: 당일진입 교체순환·현금부족 트랜치) + 구현측 보완 3건(§4).
전부 결정론(LLM 0회), 일봉 종가 조건 → 당일 종가 체결.

## 1. 공통 설정

- 초기 자본 **1억 KRW**, 무레버리지 현금계좌, 수량은 통화 기준(소수 주 허용 — 한계 명시).
- 기간: 매매 2021-01-01 ~ 2025-06-30 (분류 신호는 ~2024-12, 이후는 잔여 active pivot).
- 신호: 기존 결정론 트리거(fresh cross + 거래량 ≥ 50일평균 + **5% 추격 상한**) —
  동결 100종목 backtest_classification watch 행 기반. 이미 보유 중 종목 신호 무시,
  같은 pivot 재진입 금지(기존과 동일).
- 비용: 매수/매도 수수료 각 0.015%, 매도 증권거래세 연도별(0.23/0.23/0.20/0.18/0.15%)
  — 거래 시점 차감(기존 사전등록 §2.2 승계).
- 벤치마크: **KOSPI(주)** + KOSDAQ(보조) 동일기간 buy&hold.

## 2. 규칙 (에이전트 v1.1 확정본)

- **사이징**: `stop_pct = (entry_close − max(base_low, sma50)) / entry_close`
  (base_low 부재 시 sma50 단독). `stop_pct > 10%` → 매수 스킵(건수·분포 보고,
  스킵률 >30% 면 상류 진단 신호). `position_pct = min(1.25%/stop_pct, 25%)`.
- **슬롯**: 최대 5 포지션.
- **교체**: 슬롯 만석 + 신규 신호 → 최약(평균매입가 대비 수익률 최저) 포지션이
  **≤ 0%** 일 때만 전량 매도 후 신규 진입. 교체 후보에서 제외: (i) 당일 진입
  (ii) 8주 면제 상태. 신호 우선순위 순 연쇄 허용(면제로 유한).
- **우선순위**: rs_rating DESC(결측=최하) → 돌파일 거래량배수 DESC.
- **본전 플로어(5A, 전 시나리오)**: 종가 ≥ 평균매입가×(1+3×stop_pct) 도달 시
  armed. armed 후 종가 < 평균매입가 → 전량 청산. 기존 청산(sma50/base_low)과 OR.
- **8주 면제**: 진입(+T1) 후 21 달력일 내 종가 ≥ 평균매입가×1.20 → 진입+56
  달력일까지 교체 면제.
- **피라미딩(S2·S3)**: 목표금액 = position_pct × T1일 계좌가치(§4-②, 원화 동결),
  트랜치 50/30/20. T2: 종가 ≥ T1가×1.02, T3: ≥ T1가×1.04 최초일. 소멸: (a) 트리거일
  종가 > pivot×1.05 (b) 포지션 청산 (c) 트리거일 현금 부족(부분체결 금지).
- **강세 분할매도(5B, S3만)**: 최초 종가 ≥ 평균매입가×1.20 일에 — 진입 후 >21일이면
  50% 매도, ≤21일이면 8주 면제 진입. 진입+56일 후 첫 거래일 종가 ≥ ×1.20 이면 50%
  매도, 미만이면 5B 만료(전량 보유, 5A만). 잔여분 = 5A + 추세 청산.
- **일중 순서**: ① 청산(스톱·플로어 — 상호순서 무관, 전량 동일) → ② 교체 매도 →
  ③ 매수. 매도 대금 당일 사용 가능.

## 3. 시나리오 매트릭스 (6곡선)

| | 하락기 진입 포함 | 하락기 진입 제외* |
|---|---|---|
| S1 기본(단일진입+5A) | S1-incl | S1-excl |
| S2 = S1+피라미딩 | S2-incl | S2-excl |
| S3 = S2+5B | S3-incl | S3-excl |

*제외 = 진입일 시장국면 ∈ {downtrend, correction} 인 **신규 진입(T1)** 신호 스킵 —
트리거 확인 감사(33/33 wait) 실측 반영. 피라미딩 추가분(T2/T3)은 포지션 추종
(감사 범위는 신규 진입이므로 — 한계 명시).

## 4. 구현측 보완 3건 (design-judgment, 에이전트 v1.1 이후 확정)

- **① 매수 단계 내 순서**: 피라미딩 트랜치 먼저, 신규 진입 나중 — "승자에 더한다"
  약속 이행 우선(O'Neil 피라미딩 취지).
- **② 목표금액 동결**: position_pct × **T1 진입일 종가 기준 계좌가치**를 원화로
  동결(이후 계좌 변동 무관) — 리스크 1.25% 공식의 소급 붕괴 방지.
- **③ 최약 판정 기준가**: 교체 규칙의 수익률 = 종가/가중평균매입가 − 1 (Q5 통일 연장).
- **④ T1 현금 부족**: 신규 진입(T1) 금액보다 자유 현금이 적으면 신호 전체 스킵
  (부분 체결 금지 — 트랜치 규칙 (c)와 동일 원칙, 건수 보고). 5슬롯×최대 25%는
  자본 초과 배분이 가능하므로 이 경우가 실제 발생한다.

## 5. 산출 지표 (사전 고정)

시나리오별: 최종 계좌배수, CAGR, **계좌 MDD**(일별 equity 피크 대비), 자본 가동률
(일평균 투자금/계좌가치), 진입 수·교체 수·슬롯부족 스킵 수·10%스톱 스킵 수(+분포)·
트랜치 소멸 수(사유별)·5B 매도 수. 벤치마크 대비 초과 CAGR.

**판정 성격**: 본 시뮬은 탐색적 측정 — 사전등록 대상은 규칙·지표이며 합격선은
두지 않는다(트레이드 층에서 이미 "시장초과 미입증"). 해석 시 주의 2건 고정:
(a) 표본 100종목은 실전 신호밀도의 ~1/16 (프레임 34,022 vs 표본 2,135 분류점) —
슬롯 경합·교체 규칙은 실전보다 훨씬 덜 발동하므로 **가동률·경합 수치는 하한**,
실전 일반화 불가. (b) 우측꼬리 소수 의존·생존편향 등 기존 §8 한계 전부 승계.

# v2 — pivot 스톱 체계 (2026-07-02, v1 실행 후 결함 진단에 따른 교체. 결과 산출 전 고정)

v1 진단: 10% 스톱 스킵률 53%(중앙 13.2%) = 구조적 스톱(base_low/sma50)이 책의
초기 손절 개념과 다른 물건. 에이전트 문답 2라운드(Q1~Q6 + 보완 1~4)로 교체 확정.
규칙 대장: `docs/trading-rules-book-verified.md`.

## v2.1 청산 — 3층 스택 (구현 = 일별 유효 스톱 max, 상태 래치 없음)

```
유효 스톱(당일) = max( avg×0.92,                    # ① 초기 8% (항상)
                       avg   (armed 일 때),          # ② 본전 방어
                       sma50 (sma50 ≥ avg 일 때) )   # ③ Breakeven or Better 트레일링
종가 < 유효 스톱 → 전량 청산 (사유 = 바인딩된 최고층)
armed 조건: 종가 ≥ avg × min(1+3×stop_pct, 1.20)   # 스톱 8% → +20% (HMMS 20% 룰)
```
- 기준가(avg) = 가중평균 매입가 (초기스톱·전환·armed·+20%·5B 전부 — TLSMW ch.13).
- base_low 포지션 청산 **제거** (신호 감지 게이트에는 유지 — production 의미론).
- v1 의 sma_50 상시 청산 제거 → ③으로 대체(활성화 조건 부여). 5B·8주면제·교체·
  피라미딩 규칙은 v1.1 그대로.

## v2.2 사이징

- stop_pct = **0.08 고정** → position_pct = min(1.25%/8%, 25%) = **15.625% 일률**.
- 10% 스킵 룰 제거 → 불변식 `assert stop_pct <= 0.10` 강등 (회귀 감시).

## v2.3 검증 기준 (사전 고정 — 실현 손실 = 청산 pnl < 0 인 트레이드)

- (i′) 평균 실현 손실 ≤ **9%** (O'Neil 8% + 종가/갭 슬리피지 1%p)
- (ii) 실현 손실 중앙값 < **10%** (uncle point)
- (iii) 평균 실현 손실 < 평균 실현 이익 (trader's cardinal sin)
- 모니터링(합불 아님): 5–6% 통계목표 대비 갭, 갭 초과분(실현손실−8%) 분포,
  진입 프리미엄 3구간(0–1/1–3/3–5%, n 병기, 단조성만 해석)별 스톱아웃률·평균손익,
  피라미딩/단일 트랜치 스톱아웃률 분리.

# v3 — 시장 게이트 3-arm 실험 (2026-07-03, 결과 산출 전 고정)

에이전트 자문(하락장 게이트 설계 공간, 5R) + 승인 조건 3건 반영. production 무접촉
(변형 국면은 백테스트 로컬 재계산 — market_context_daily 저장 이력 + index_daily).

## v3.0 기존 수치 측정조건 주석 (승인 조건 ①)

**기존 excl 모드(v1·v2의 "실전 재현")는 실제 production 게이트보다 느슨했다**:
진입일 국면 ∈ {downtrend, correction}만 차단하고 rally_attempt 진입을 허용했으나,
실제 트리거 확인(5b §3.5 분기)은 watch_reason=unfavorable_market 에 대해
**rally_attempt 도 wait** 다(감사 33건 중 29건이 해당 사유). 따라서 기록된
v2 S1-excl CAGR +1.49%·기회비용 ≈연 7.6%p 는 **하한이 아니라 느슨한-게이트
측정치**이며, 진짜 재현은 더 낮을(기회비용은 더 클) 가능성이 높다.
이후 모든 비교의 기준선은 v3 Arm A 로 교체한다.

## v3.1 3-arm 정의 (승인 조건 ②) — 전부 v2 스톱 체계 + S1 구성, 동일 데이터

| Arm | 게이트 | 국면 소스 |
|---|---|---|
| **A 정밀 production 재현 (신규 기준선)** | T1 신호의 watch_reason=unfavorable_market 이고 국면 ≠ confirmed_uptrend → 스킵. 그 외 사유는 통과 | 현행 사다리 (market_context_daily 저장값) |
| **B 변형 사다리 (c)** | A 와 동일 게이트 의미론 | 변형 사다리 (v3.2) |
| **C 레거시 (연속성 확인용)** | 국면 ∈ {downtrend, correction} → 전부 스킵 (v2 excl 그대로) | 현행 사다리 |

한계 명시: Arm A 는 §3.5 의 코드화 가능한 분기(unfavorable_market)만 재현.
감사에서 marginal_tt·valid_base 4건도 wait 였으므로(표준 검증 사유로 추정)
실제 게이트는 A 보다 더 엄격할 수 있음 — A 는 게이트 강도의 하한.

## v3.2 변형 사다리 (옵션 c) — 결정론 스펙 (승인 조건 ③)

현행 6규칙에서 규칙 4·5만 교체, **우선순위 1→6 보존** (규칙 1·2·3 이 4' 보다
상위 — confirmed 고아 경로 없음, 테스트로 확인):

- 규칙 4′ (confirmed_uptrend): **유효한 FTD 존재 AND dist_count < 6**.
  현행의 `close > SMA50` 대기와 `90일 시간창` **제거** (책에 없는 2차 게이트).
- FTD 유효성: 발생일부터 **랠리 저점 종가 이탈 시까지**.
  - **랠리 저점 = FTD 일 포함 직전 15세션의 최저 low** (FTD 는 저점 후 3~15세션
    발생이므로 이 윈도가 랠리 기산 바닥을 포함). [무효화=신저가 회귀 개념: book /
    앵커·윈도 수치: design-judgment]
  - **이탈 판정 = 지수 종가 < 랠리 저점** (장중 아님 — 노이즈 무효화 방지,
    design-judgment).
  - FTD 이력 복원: market_context_daily.last_follow_through_day 는 90일 창
    내에서만 기록되므로, 시계열 carry-forward 로 전 이력 복원(각 FTD 이벤트는
    발생 후 90일간 컬럼에 존재 → 누락 없음).
- 규칙 5′: close > SMA50 AND FTD 없음/무효화됨 → rally_attempt (시간 조건 제거).
- 지수 SMA50/200·52주 고점 대비 하락률은 index_daily 종가로 재계산(현행 정의 동일).

# v4 — 2단계 bottoming_attempt 파일럿 (2026-07-03 **DRAFT — 잠금 전 검토용**)

## v4.0 목적 (가치 명제의 좁힘 — 승인 조건 반영)

v3 한계분해에서 016710 꼬리는 **현행 게이트(Arm A)도 포착**함이 확인됐다. 현행이
놓치는 꼬리는 "모든 대박"이 아니라 **깊은 바닥 전환 창 유형(앵커: 004360 2022-10·
053350 2025-01)**으로 좁혀졌다. 본 실험은 정확히 그 창 — 시장 라벨이 down/
correction 이고 게이트가 차단 중이나 지수가 랠리 시도 중인 구간 — 에서 Minervini
파일럿(TLSMW ch.13)이 값을 내는지 검증한다. **순환성 주의**: 파일럿 규칙 자체가
앵커 2건을 보고 설계된 가설이므로, 합격 판정은 앵커 제외 기준(v4.4)이 필수다.

## v4.1 bottoming_attempt 정식 스펙 (드라이런 근사 확정)

- 지수(시장별) 기준: **15세션 신저가 발생 → 이후 첫 상승 마감일부터 랠리 카운트
  활성 → 레그 저점(신저가일 low) 종가 이탈 시 리셋**. 신저가 갱신 시 레그 교체.
- O'Neil 변형(신저가일 종가가 당일 레인지 중간 이상이면 당일 기산)은 **미포함**
  — design-judgment, 단순성 우선(포함해도 무방하나 자유도 축소 우선).
- bottoming_attempt(d) = 현행 사다리 라벨 ∈ {downtrend, correction} AND 랠리
  카운트 활성. 파일럿 허용 = 랠리 카운트 ≥ 1 (Minervini stock-first, PCYC 근거).

## v4.2 파일럿 규칙 (전부 단일 값 — 범위 금지)

- **대상**: Arm A 게이트가 차단하는 신호(unfavorable_market ∧ 비confirmed) 중
  진입일이 bottoming_attempt 인 것. 그 외 신호 처리는 Arm A 와 동일.
- **사이즈**: 정상 목표(15.625%)의 **50% = 7.81%** (계좌 대비). 스톱 6% 와 결합 시
  건당 계좌 리스크 ≈ 0.47% (정상 1.25% 의 ~3/8) — TTLC "어려운 환경에서 사이즈·
  노출 축소" 정합. [50%·아래 6% 모두 design-judgment]
- **스톱**: 파일럿 포지션 초기 스톱 **6%** (avg×0.94). 본전 방어 armed 임계는
  min(3×0.06, 0.20)=+18%. 나머지 청산 스택(플로어·sma50 트레일)·비용 동일.
- **재시도 상한**: 같은 (종목, bottoming 에피소드) 내 **최대 2회 진입**
  (스톱아웃 후 재신호 1회까지 허용 — TLSMW "반복 손절 = 너무 이름" 정합).
  에피소드 = 레그 저점 날짜로 식별되는 연속 구간.
- **증액 트리거 (결정론화)**: 다음 중 하나가 성립하는 날부터 **신규 진입을 정상
  사이즈(15.625%·스톱 8%)로** 전환(기존 파일럿 증액은 안 함 — 단순성):
  (i) 해당 시장 FTD 유효(발생~랠리저점 종가이탈 전, v3.2 정의 재사용), OR
  (ii) 활성 파일럿 ≥ 2 AND 파일럿군 합산 미실현 손익 > 0.
  두 조건 모두 일별 재평가(래치 없음 — 조건 소멸 시 파일럿 사이즈로 복귀).
