## 3. 질문서 (web claude 에 붙여넣기)

```
너는 Minervini(TLSMW/TTLC)·O'Neil(HMMS)·Morales&Kacher(TLOND) 원저 PDF 를 보유한
프로젝트다. 첨부한 것은 한국 주식 주간 분류 시스템의 LLM 프롬프트 전문
(analyze_chart_v3.md)과, 이 시스템이 실제로 분류한 종목 입력 샘플 6건(텍스트+차트)이다.

질문 범위는 **책-충실성으로 한정**한다 — 일반적 결함·문체·구조 제안은 하지 말 것
(내부 감사와 중복). 다음만 답하라:

1. 임계값 정합: 프롬프트의 SSOT 블록·본문 임계(컵 깊이 33/50%, 선행상승 30%,
   핸들 8-12%·1주+, climax 25%/3주·18주/12주 성숙, topping 10주선 8주·분배일 4,
   시장 분배일 5, marginal 3%/3개, ±5% 밴드 등)가 책 원문(페이지 인용)과 일치하는가?
   어긋나면: 책 원문 인용 + 어느 방향으로 얼마나.
2. 패턴 정의 정합: §4 패턴 표(8종)의 정의·pivot 산출(§4.7)이 책과 일치하는가?
   특히 **F7 — cup-without-handle**: 책(HMMS)은 손잡이 없는 컵을 정식 패턴(매수점=
   컵 좌측 고점)으로 인정하는데 이 taxonomy 에는 없다. 책 기준으로 (a) 별도 enum 이
   필요한가 (b) 어느 정의(매수점·최소 주수·깊이)로 넣어야 하는가?
3. 게이트 로직 정합: Gate0~3 측정-우선 트리, §6.1 climax(P1/P2/T1-4/E1),
   §6.2 topping(G0/T-A~D)의 구조가 책의 판정 순서·조건과 부합하는가?
   책에 없는 시스템 자체 확장(EXTENDS)이 책 취지와 충돌하는 지점이 있는가?
4. 샘플 6건 각각: 이 입력에서 책 기준으로 verdict(entry/watch/ignore)와 그 근거
   조항은 무엇인가? (시스템의 실제 판정은 검토가 끝난 뒤 공개한다 — 앵커링 방지)

각 지적마다: [책 원문 인용(저작+페이지)] / [프롬프트 현행 문구] / [제안] 형식.
확인이 필요한 시스템 동작은 단정하지 말고 질문으로 남겨라(우리가 리포에서 확인 후 답한다).
```
