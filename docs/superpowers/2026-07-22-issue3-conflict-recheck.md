## 이슈 1 — 손절 anchor (pivot vs 매수가)

- **위치 이동**: 프롬프트 §2 는 은퇴(#21) → `kr_pipeline/llm_runner/compute/entry_params_calc.py`
  §2 (:168-211)가 대체. **산식은 동일 이식**: anchor = pivot
  (`stop_price = pivot × (1+stop_pct/100)`, :208), 클램프 `[ENTRY_STOP_PCT_FROM_PIVOT_FLOOR(−10), −5]`
  (:204), 기본 absolute −7.0(표준)/−5.5(tight). 현재가 기준은 `|stop_from_current| > 7.5%`
  경고만(:210-211, `stop_distance_from_current_price_exceeds_book_limit`) — 차단 아님.
- **#45 효과 (발현 범위 축소)**: 원 조사의 발현 조건 "추격 +5% *초과* 진입"은 PR #66 의
  결정론 extended 게이트(close > pivot×1.05 → go_now 불가)가 **원천 차단** — 최악 케이스
  (−14~15% 초과분)의 꼬리가 잘렸다.
- **잔존 상충 (리뷰 정정 반영)**: 스톱은 `max(logical, absolute)` 선택(:195-198)이라
  **absolute(−7.0 표준/−5.5 tight)보다 나빠질 수 없다** — "−10% binding → −14.3%"
  시나리오는 도달 불능(코드 주석 :201-203 명시). 진짜 최악 = buy zone 상단
  (pivot×1.05) 체결 + absolute −7.0 → 매수가 기준 **−11.43%** (uncle point 초과분
  ~1.4%p). 그마저 VCP 는 chase 상한 3.0%라 −9.7%로 **uncle point 안쪽** — 초과는
  chase 5% 허용 패턴이 밴드 최상단(+3.3% 초과 프리미엄)에 체결된 좁은 구석에서만.
  규율대로 trigger_price(pivot×1.001) 체결이면 ≈−7.1%로 여유. 즉 조건부 상충은
  형식적으로 남아 있으나 발현 폭·확률 모두 원 조사보다 크게 작다.
- **수정 시 절차**: `ENTRY_STOP_*` 소비 로직 변경 = threshold-change-checklist 의존성 맵
  대상. anchor 를 매수가로 바꾸는 안은 C단계 재설계(사전등록) — 원 이슈 처리 방침 2번 유지.

## 이슈 2 — 사이징 방법론 (티어 vs 리스크 역산)

- **위치 이동만**: `entry_params_calc.py` §3 (:213-253) — 셋업 품질 티어(3~25%,
  `ENTRY_WEIGHT_PCT_MIN/MAX`) + 플래그 감산. **스톱 거리와 무관한 구조 그대로** —
  Minervini 리스크 역산(계좌 1.25%/건 ÷ 스톱폭)은 여전히 미구현. 방법론 상충 불변.
- 참고: 이슈 1 을 매수가 anchor 로 수정하는 경우 스톱폭 정의가 바뀌므로, 역산 사이징
  도입은 이슈 1 결정과 **함께** 설계하는 것이 순서다(따로 하면 두 번 갈아엎음).

## 이슈 3 — base_low 재검토 신호의 둔감성

- **현행 동일**: `load.py:164` `stop_loss = base_low` → `trigger_gate.evaluate` invalidation
  (`close < stop_loss OR close < sma_50`). 변경 이력 없음 — 둔감성(실측: pivot −24.6% 에도
  미발동 사례) 그대로.
- **안전장치 승계 완료**: 원 조사의 진짜 우려("이 정의를 실포지션 매도에 재사용")는
  #47 이슈 코멘트(2026-07-22)와 스펙 §3 불변 계약에 **재사용 금지로 명시** — 밤 2
  구현이 준거로 삼는다. 재검토 신호 자체의 둔감성 개선은 별도 후보로 남김(비긴급 —
  용도가 관찰 트리거라 실손실 직결 아님).

## 이슈 4 — 포지션 관리 부재 → 절반 해소

- **코어 완료**: 3층 손절 스택은 PR #40 이 `kr_pipeline/trade_management/stop_stack.py`
  로 이식(준거 구현과 산술 비트 동일 검증, SSOT 상수 3종 = TRADE_STOP_INITIAL_PCT 0.08 ·
  TRADE_BREAKEVEN_TRIGGER_PCT 0.20 · TRADE_STOP_MAX_PCT 0.10).
- **잔여 = wiring**(#47): positions 테이블·일일 평가 러너·신호 노출 — 소비자 0 인 additive
  상태. 선행 결정(포지션 소스 = **수동 기록**) 2026-07-22 확정.
- **긴급도 상승**: 2026-07-21 실전 전환으로 go_now 발생이 가능해짐 — 매수 신호만 있고
  매도 관리가 없는 반쪽 상태. **#47 완료 전 실제 자금 투입 금지**를 명시적 게이트로 권고.

## 처리 방침 갱신 (원 이슈의 dry-run 전제 소멸)

1. **#47 (이슈 4 wiring) 최우선** — 진행 중(밤 2), 완료 전 실매매 개시 금지.
2. 이슈 1+2 는 **한 묶음의 사전등록 후보**로 승격(anchor 변경과 역산 사이징은 상호 의존)
   — 착수 시 threshold-change-checklist 의존성 맵 필수. 즉시 강제는 아님: #45 가 +5%
   초과 꼬리를 잘랐고 max() 선택 구조가 스톱을 absolute 로 바닥 지지해, 실질 초과분이
   좁은 구석의 ~1.4%p 에 그친다(리뷰 실측). 발현은 go_now 실적이 쌓인 뒤 실측으로
   판단 가능(#45 사전등록 코호트와 같은 데이터).
3. 이슈 3 은 #47 체크리스트로 이관 완료 — 본 이슈에서 종결 가능.
