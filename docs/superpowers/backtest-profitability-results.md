## ⚠️ 범위 라벨 (먼저 읽을 것)

핵심 질문은 "얼마 버나"가 아니라 **"하락·조정기에 게이트가 진입을 억제하나"**(§0).
트레이드 층은 **결정론 트리거의 하한(lower-bound) 추정**이다 — LLM 트리거 확인·포지션
사이징 제외, `breakout_from_watch`가 시장 비게이팅이라 **하락기 진입을 과대평가**(§4,
보수적 방향). 게이트 방어의 1차 판정은 트리거 근사에 오염되지 않는 **분류층**(§7.1)이다.

## §7.1 게이트 방어 (1차, 사전등록) — **PASS**

| 지표 | 값 |
|---|---|
| R_down = entry-rate(downtrend ∪ correction) | **0 / 1,075 = 0.0** |
| R_up = entry-rate(confirmed_uptrend) | 1 / 292 = 0.0034 |
| R_down / R_up (기준 ≤ 0.5) | **0.0 → PASS** |

하락·조정기 분류점 1,075개에서 `entry` 분류 **정확히 0건**. §3.5 hard rule 이 스펙의
기대("R_down은 0에 가까울 것") 그대로 작동했고, 유일한 entry 1건도 confirmed_uptrend
에서 나왔다. **게이트의 하락기 진입-분류 억제 = 입증.**

읽기 주의: entry verdict 자체가 극도로 희소(1/2,135)하므로 이 판정은 "직접-entry 경로의
방어"다. 실제 매수 노출은 watch→트리거 경로가 지배하며, 그 경로의 하락기 노출은 아래
§7.2 트레이드 층(비게이팅 근사)으로 본다.

## §7.2 손실 억제 — 시장대비 초과수익 (보조) — **supportive 아님 (게이트 실패 아님)**

트레이드 66건(동결 100종목 production 시뮬, 진입일 국면 라벨):

| 국면 | n | 평균 초과 | 중앙 초과 | 승률(초과>0) | 평균 P&L |
|---|---|---|---|---|---|
| downtrend | 9 ⚠️underpowered | **+25.6%** | −2.5% | 44.4% | +23.1% |
| correction | 24 | **−7.3%** | −6.4% | 16.7% | −6.6% |
| rally_attempt | 21 | +7.3% | −3.9% | 23.8% | +5.8% |
| confirmed_uptrend | 12 | −3.3% | −8.6% | 25.0% | −2.5% |

- 사전등록 기준(하락∪조정 평균 초과 ≥ 0)은 **correction −7.3%로 미충족** → supportive
  아님. 사전등록 오판-방지 조항 그대로: **§7.2 음수 ≠ 게이트 실패** — 이 트레이드들은
  시장 비게이팅 트리거(§4)가 강제한 진입을 포함하므로 "결정론 근사의 하한 손실"로
  해석하고, 게이트 판정은 §7.1(분류층, PASS)로 한다.
- 하락·조정기 트레이드 33건(66의 50%)은 전부 watch pivot 의 `breakout_from_watch`
  발화 — 실제 시스템은 LLM 트리거 확인에서 §3.5를 재적용하므로 실전 하락기 노출은
  이보다 적다(하한 방향).
- downtrend 평균 +25.6%는 대형 아웃라이어 2건(004360 2022-10 진입 +159.9%,
  053350 2025-01 진입 +99.6% — 약세장 바닥 진입 후 회복 편승)이 견인. 이 2건 제외
  평균 −4.1%, 중앙값 −2.5%. n=9 < 10 이라 **§7.5 underpowered — 과해석 금지**.
- correction 24건의 분포는 방어적 실패 패턴이 아니라 whipsaw 패턴: 24건 중 20건이
  −22%~−1% 구간의 통제된 손절, 최악 1건 −46.9%(077500, base_low 청산).

## §8 한계 (사전 명시분 재확인)

1. **결정론 트리거·청산 = 하한 추정** — LLM 트리거 확인·사이징 제외.
   `breakout_from_watch` 시장 비게이팅으로 하락기 진입 과대평가(§7.2가 그 영향권).
2. **입력 분류 = 1회 LLM 백필** — 비결정성 때문에 재실행 비교 금지, "그 입력 고정
   하의" 결과.
3. **생존 편향 잔존** — 상폐 종목 부재(DB delisted_at 전무), 결과가 부풀려지는 방향.
   우측 꼬리(+222% 등)는 특히 이 편향의 수혜 가능.
4. **진입일-기준 국면 라벨** — 보유 중 국면 변동 미반영. downtrend 아웃라이어 2건이
   정확히 이 케이스(하락기 진입 → 회복기 보유).
5. **KR 단일 시장·2021–2024** — 타 시장/기간 일반화엔 추가 표본.
6. (실행 중 추가) 가격 윈도가 2025-06-30까지라 2024 말 watch 에서 2025년 진입
   트레이드 7건 포함, 그중 3건은 윈도 끝 미청산(open) — 국면 라벨·집계엔 포함되나
   최종 P&L 은 절단값.

### 개선 1. 하락기 진입 33건 LLM 트리거 확인 감사 — **G = 0.000, PASS**

1차 결과의 "하한" 주장을 실측으로 검증: 시뮬이 만든 하락·조정기 진입 33건 각각을
production 트리거 확인(5b, §3.5 재적용)으로 과거 시점 재생(prior = 해당
backtest watch 행 주입, 미래 날짜 토큰 0 검증 후 실행).

- **33건 전건 `wait`** (go_now 0, abort 0). G = 0/33 = **0.000 ≤ 0.5 → PASS**.
- watch_reason 분해: unfavorable_market 29(프롬프트 규칙 기대 그대로 전건 wait),
  marginal_tt 2·valid_base_awaiting_breakout 2 도 전건 wait.
- **함의**: 실전 시스템의 하락·조정기 진입 추정 = 33 × 0.0 = **0건**. 1차 §7.2 의
  correction −7.3% 손실 트레이드들은 실제 시스템에서는 발생하지 않았을 것.
  **동시에 하락기 대형 승리 2건(004360 +160%·053350 +100%)도 차단** — 게이트는
  양쪽 꼬리를 모두 자른다(방어는 수익 포기와 교환).
- 기록: `data/backtest/trigger_audit_20260702.json` (전건 llm_model=claude-sonnet-5,
  production trigger_evaluation_log 쓰기 0건). LLM 1회 실행 — 재실행 비교 금지.

## 결론 (개선 1~3 반영)

- **방어(사전등록 §7.1 + 감사): 두 층 모두 완벽 작동 입증.** 분류층 R_down=0/1,075,
  트리거 확인 층 G=0/33. 실전 시스템의 하락·조정기 매수 노출은 분류·확인 양층에서
  0 으로 실측됐다 — 1차의 "하한" 유보가 실측으로 해소.
- **수익(§2.5·§3 사전등록 기준): 둘 다 미입증.** 평균 초과 +3.2%는 95% CI 가 0 을
  포함하고, 플라시보 대비 p=0.19. correction·confirmed_uptrend 국면은 유의하게
  음수. "잃지 않는 시스템"은 입증됐으나 "시장을 이기는 시스템"은 이 표본으로
  입증되지 않았다.
- 수익성 판정에는 여전히 표본이 부족(우측 꼬리 소수 의존) — 기간 확장(2016~2020)
  또는 포트폴리오 단위 시뮬이 다음 단계 후보.
