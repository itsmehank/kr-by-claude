# 후속 질문 — 제안 규칙 적용 전 4가지 확정 필요

이전 답변(climax §6.1 게이트, 규칙2 예외, late_stage 4th+, extended informational,
재진입 셋업 2종) 거의 그대로 적용 예정이다. 적용 전, 구현을 막거나 약화시키는
4가지를 확정하고 싶다. 모두 repo 의 `prompts/analyze_chart_v3.md` 와 책을 기준으로
답해 달라.

## 참고: LLM 이 실제로 받는 데이터 (규칙 작성 시 전제)
- 주봉 OHLCV **raw 104주** (사전계산 주봉 MA 는 미제공 — 10주 등 MA 는 종가로 직접 계산)
- 일봉 OHLCV 최근 60일 + 일봉 차트 PNG(1년) + 주봉 차트 PNG(2년)
- 일봉 지표: SMA 10/21/50/150/200, 52주 고저, RS rating, 거래량비
즉 규칙은 위 데이터로 *계산 가능*해야 한다.

## Q1. "advance began" 앵커의 조작적 정의 (§6.1 전체의 토대)

§6.1 의 P1(성숙도 ≥18/12주), P2("전체 상승 내 최고 가속"), T1~T4("since the advance
began") 가 모두 **"현재 상승의 시작점"** 을 기준으로 한다. 그런데 LLM 이 주봉 차트에서
이 시작점을 어떻게 식별하는가?

- 이 앵커가 Q2(late_stage)의 base-count 앵커("종목 자신의 Stage 4 / 깊은 조정 이후
  새 advance 시작")와 **동일한 점**인가? (그렇다면 SK하이닉스의 advance 시작 = 2025-06
  최초 컵 돌파, 전체 multi-base 런이 하나의 advance)
- LLM 이 적용할 수 있는 **판별 규칙**을 한 줄로 — 예: "advance start = 직전에 종목이
  30주선/40주선 아래로 깨지고 Stage 1 베이스를 만든 뒤 거래량 동반 돌파한 주"?
- 이 앵커가 차트 창(주봉 104주) 밖이면 어떻게 처리하나?

## Q2. P1 성숙도 18주 / 12주 — 책 숫자인가 보간인가

답변은 이 수치를 "O'Neil HMMS pp.262-263, exhaustion-gap timing" 으로 인용했다.
이 프로젝트는 임계의 출처를 (a) 책 *명시 숫자*(PRESERVES) vs (b) 책은 개념만 제시,
숫자는 시스템 채택(EXTENDS) 으로 엄격히 구분해 기록한다(SSOT 규율).

- 18주(1·2차 베이스 후) / 12주(후기 베이스 후) 가 HMMS p.262-263 에 **literal 로 적힌
  숫자**인가, 아니면 exhaustion-gap 논의로부터의 합리적 보간인가?
- literal 이면 정확한 표현/페이지를, 보간이면 그 사실을 명시해 달라
  (우리는 보간이면 "design-judgment, community/expert convention" 으로 라벨한다 —
  숨기지 않기 위함).

## Q3. E1 제외 규칙의 적용 범위 — false-negative 위험

E1: "≥25% 상승이 valid base breakout 후 3주 이내면 climax 아닌 leadership"
(근거: HMMS p.269 8주 보유 룰).

- p.269 8주 보유 룰은 **최초 돌파(첫 proper base 의 pivot)** 맥락인데, E1 을
  **모든 베이스 돌파**(2·3·4·5차 포함)에 적용하는가, 아니면 **최초 돌파에만**?
- 만약 모든 돌파에 적용한다면: 답변이 "진짜 climax" 로 지목한 2026-05 클러스터가
  5차 베이스 돌파 직후 3주 이내라면 E1 에 걸려 climax 가 **제외**된다 —
  유일한 진짜 climax 를 놓치는 모순. 이 경우 E1 과 §6.1 의 우선순위를 어떻게 두나?
- 후기 베이스(4·5차) 돌파 직후의 급등은 leadership 인가 late-stage climax 위험인가?
  (HMMS p.268 "4차 베이스 80% 실패" 와 p.269 "leadership 팔지 마라" 의 경계)

## Q4. 개선 성공 기준 — 책 기준 기대 분포

재백필로 효과를 정량 검증하려 한다. "watch 가 늘면 좋다" 는 검증 불가하므로,
**책 기준의 목표 분포**가 필요하다.

- 제안 규칙 적용 후, SK하이닉스 같은 슈퍼 주도주의 52주 분류는 대략 어떤 분포여야
  책-정합인가? (예: watch ~N주(베이스 형성·돌파대기 구간), ignore ~4-5주(진짜 climax),
  entry 는 주간 스냅샷 특성상 0~소수)
- "유효 피벗이 존재한 주" 중 watch 로 포착돼야 하는 비율의 합리적 목표치는?
  (백테스트 KPI 정의에 쓰겠다 — 단, LLM 출력과 독립적인 *책 기준* 정의가 필요)
- 위 5건 돌파(06-14, 09-13, 09-20, 10-04, 01-24)는 전부 watch 로 잡혔어야 하는가,
  아니면 그중 책 기준상 watch 가 부적절한(예: 너무 extended) 케이스가 있는가?
